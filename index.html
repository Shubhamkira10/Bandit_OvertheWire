Overview

Welcome to the OverTheWire Bandit Challenge Solutions Page! This guide offers step-by-step solutions and detailed explanations for each level of the Bandit Challenge. Whether you're a beginner learning Linux commands or an experienced user sharpening your skills, this resource will help you understand the logic behind each challenge and improve your command-line proficiency.

Commands

Here are some essential commands used throughout the Bandit challenges:

  • sshpass -p "password" ssh user@host -p 2220 : SSH login without interactive password entry
  • ls : List files in a directory
  • cat filename : Display the contents of a file
  • cat "$(ls)" : use command substitution to read a file
  • cat -- "filename" : use -- to handle special filenames
  • pwd : Print the current directory
  • cd directory : Change directory
  • file filename : Determine the file type
  • ls -a : List all files, including hidden ones.
  • ls -la : Show detailed information about all files, including hidden ones.
  • ls -d .* : List only hidden files.
  • |(pipe) : It is used in Linux to redirect the output of one command as input to another command. This allows you to chain multiple commands together and process data efficiently.
  • find . -type f : Find all files in the directory.
  • find . -type f | xargs file: finds all regular files in the current directory and checks their type using the file command.
  • find . -name "filename" -exec cat {} \; : find a file and read its contents
  • grep: (Global Regular Expression Print) command is used in Linux to search for specific text patterns inside files or command outputs.
  • sort: The sort command sorts lines of text in a file alphabetically or numerically
  • uniq: The uniq command removes or filters duplicate lines in a file.
  • strings: it is used to extract readable text from binary files, such as executables (.exe, .bin), libraries (.so), or any other non-text files.
  • base64 -d : it is used to decode data that has been encoded in Base64 format.
  • tr (translate): command is used to translate, delete, or squeeze characters from the input. It reads from standard input (stdin) and writes to standard output (stdout).
  • gzip : Compresses files using the GNU zip format (.gz).
  • bzip2 : Compresses files using the Burrows-Wheeler block sorting algorithm (.bz2).
  • tar -xf :Extracts files from a tar archive (.tar, .tar.gz, .tar.bz2).
  • xxd : Creates a hexdump of a file or converts a hexdump back to binary.
  • mv: Moves or renames files and directories.
  • cp: Copies files and directories.
  • mktemp -d : Creates a temporary directory with a unique name.

For this level, you don’t get the next password, but you get a private SSH key that can be used to log into the next level. Command to log in:

Level 14 → Level 15

Goal: The password for the next level can be retrieved by submitting the password of the current level to port 30000 on localhost.

Command to read the current level's password

bandit14@bandit:~$ cat /etc/bandit_pass/bandit14

To retrieve the password for the next level, you need to send the current level's password to port 30000 on localhost. You can do this using the nc (Netcat) command:

bandit14@bandit:~$ nc localhost 30000
bandit14@bandit:~$ telnet localhost 30000
bandit14@bandit:~$ curl telnet://localhost:30000

Command to log in:

Since port 30001 requires SSL/TLS encryption, you need to use OpenSSL instead of Netcat.To retrieve the password for the next level, you need to send the current level's password to port 30001 on localhost

bandit15@bandit:~$ openssl s_client -connect localhost:30001

uses Ncat, a powerful networking utility similar to Netcat but with added features like SSL/TLS support.

bandit15@bandit:~$ ncat -C --ssl localhost 30001

Goal: The credentials for the next level can be retrieved by submitting the password of the current level to a port on localhost in the range 31000 to 32000. First find out which of these ports have a server listening on them. Then find out which of those speak SSL/TLS and which don’t. There is only 1 server that will give the next credentials, the others will simply send back to you whatever you send to it.

Command to log in:

Once you have the list of open ports, check which ones support SSL/TLS. You can use ncat or openssl for this:

bandit16@bandit:~$ openssl s_client -connect localhost:"CORRECT_PORT -quiet

Copy the RSA PRIVATE KEY and paste in a file "sshkey"

bandit16@bandit:~$ cd $rough
bandit16@bandit:/tmp/tmp.ms3pGvPn0x$ vi sshkey

press "Insert" button and paste the RSA PRIVATE KEY, then press "Esc" button and type ":wq"
